# 產品需求文件（PRD）

## 1. 文件資訊
- **產品名稱**：專案管理甘特圖軟體（暫名：GanttFlow）
- **版本**：v0.1（審閱版）
- **文件日期**：2026-04-25
- **文件狀態**：待審閱
- **目標讀者**：產品經理、設計師、前後端工程師、測試工程師、營運/管理者

## 2. 產品背景與目標
許多中小型團隊仍以試算表追蹤專案進度，常見痛點包括：
1. 任務相依關係不易可視化。
2. 時程變更後難以快速更新全局計畫。
3. 對外報告需要手動整理成不同格式。

本產品旨在提供「快速建立專案排程 + 甘特圖可視化 + 一鍵匯出 Excel/PDF」能力，降低專案管理與彙報成本。

### 2.1 產品目標（MVP）
- 使用者可建立專案與任務，並在甘特圖上檢視時程。
- 支援任務相依關係與里程碑。
- 使用者輸入資料後可匯出為 **Excel (.xlsx)** 與 **PDF (.pdf)**。
- 匯出內容可作為管理會議/客戶報告直接使用。

### 2.2 非目標（MVP 不含）
- 資源容量最佳化排程（AI 自動排程引擎）。
- 即時多人協作編輯（先以單使用者或低衝突協作為主）。
- 手機 App 原生版本（先做 Web 響應式）。

## 3. 目標使用者與角色
1. **專案經理（PM）**：建立排程、追蹤進度、輸出週報。
2. **部門主管**：檢視跨專案里程碑與風險。
3. **執行成員**：更新任務狀態與實際工時。
4. **外部利害關係人（客戶）**：讀取 PDF 報告，不一定登入系統。

## 4. 使用情境（User Stories）
1. 身為 PM，我希望快速輸入任務與日期，立即看到甘特圖，以便確認時程是否衝突。
2. 身為 PM，我希望設定任務相依關係，避免下游任務提前開始造成風險。
3. 身為主管，我希望一鍵輸出 PDF，於會議中直接簡報。
4. 身為 PM，我希望輸出 Excel，進一步做內部分析與備份。

## 5. 功能範圍

### 5.1 MVP 核心功能
- 專案建立/編輯/封存。
- 任務 CRUD（新增、查詢、更新、刪除）。
- 甘特圖視圖（日/週/月切換）。
- 任務相依關係（FS 為主：Finish-to-Start）。
- 里程碑（Milestone）節點。
- 進度追蹤（% complete）。
- 匯出 Excel。
- 匯出 PDF。

### 5.2 後續版本候選
- 基準計畫（Baseline）與差異分析。
- 資源負載視圖。
- 權限細分（RBAC 進階）。
- 與日曆/工單系統整合。

## 7. 主要流程

### 7.1 建立與編輯排程
1. 使用者建立專案（填寫專案層級欄位）。
2. 新增任務並設定起訖日期、負責人、進度。
3. 設定任務相依關係（FS）。
4. 系統繪製甘特圖並即時反映變更。

### 7.2 匯出流程（Excel / PDF）
1. 使用者點擊「匯出」。
2. 選擇匯出格式（Excel 或 PDF）。
3. 設定匯出參數（日期區間、是否含完成任務等）。
4. 系統進行資料驗證與格式生成。
5. 下載檔案並記錄匯出歷程（操作人、時間、條件）。

## 8. 匯出規格

### 8.1 Excel 匯出（.xlsx）
- 工作表建議：
  - `Project_Summary`：專案基本資訊、整體進度、關鍵里程碑。
  - `Task_List`：任務明細（含依賴欄位、進度、工時）。
  - `Timeline`：以欄位色塊呈現簡化甘特時間軸。
- 格式要求：
  - 日期欄位為真正日期格式（非純文字）。
  - 進度為百分比格式。
  - 凍結首列、啟用篩選。

### 8.2 PDF 匯出（.pdf）
- 內容包含：
  - 專案摘要（名稱、期間、狀態、負責人）。
  - 甘特圖主視圖（所選日期區間）。
  - 里程碑清單與關鍵風險。
- 視覺要求：
  - 支援 A4/A3 與橫式。
  - 需加上頁首（專案名稱）與頁尾（頁碼、匯出時間）。

## 9. 驗證規則與錯誤訊息
- 日期錯誤：結束日早於開始日，阻擋提交並提示具體欄位。
- 相依循環：偵測出 A→B→A 時阻擋儲存。
- 必填缺漏：於欄位下方顯示錯誤訊息。
- 匯出失敗：提供錯誤代碼與「重試」按鈕。

## 10. 非功能需求
- **效能**：1 個專案 1,000 筆任務時，甘特圖首次載入 < 3 秒（快取後）。
- **可用性**：核心操作（新增任務、拖曳調整、匯出）在 3 步內完成。
- **相容性**：支援最新版 Chrome、Edge、Safari。
- **安全性**：匯出檔需遵守權限控管，不得越權下載。
- **稽核性**：保留匯出紀錄（操作者、時間、篩選條件）。

## 11. KPI 與驗收標準

### 11.1 KPI（MVP 上線後 3 個月）
- 70% 活躍專案至少匯出過 1 次報表。
- 50% PM 每週至少更新一次進度。
- 匯出成功率 >= 99%。

### 11.2 驗收標準（UAT）
1. 能新增專案並新增至少 20 筆任務成功顯示於甘特圖。
2. 能設定任務相依關係並阻擋循環依賴。
3. 能匯出 Excel 且欄位完整、格式正確。
4. 能匯出 PDF 且甘特圖與摘要可閱讀。
5. 匯出時間在 10 秒內（100 筆任務基準）。

## 12. 風險與待確認項目
- 是否需要多語系（繁中/英文）同時上線。
- PDF 是否需企業識別（Logo、品牌色、浮水印）。
- Excel 模板是否需客製（依客戶不同欄位）。
- 權限模型是否需區分「可看不可匯出」。

## 13. 里程碑建議（MVP）
- W1-W2：需求確認、資訊架構、欄位定義凍結。
- W3-W4：任務 CRUD + 甘特圖基礎渲染。
- W5：相依關係與驗證規則。
- W6：Excel/PDF 匯出。
- W7：整合測試與 UAT。
- W8：上線與教育訓練。
